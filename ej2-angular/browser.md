# Browser Compatibility in Angular

Syncfusion<sup style="font-size:70%">&reg;</sup> Angular UI components are designed to work seamlessly with modern web browsers. This document outlines supported browser versions and provides key information on compatibility.

## Supported Browser Versions

The following table lists the minimum supported versions for various browsers:

| Browser | Supported Versions |
|:--------|:-------------------|
| Google Chrome, including Android & iOS  | Latest 2 versions |
| Mozilla Firefox	 | Latest version |
| Microsoft Edge	    | Latest 2 versions |
| Apple Safari, including iOS	  | Latest 2 versions |


> Note: "Latest" refers to the most recent stable version of the browser.

For the most up-to-date information, refer to the official Angular documentation on [Supported Browsers](https://angular.dev/reference/versions#browser-support).

## Modern Web Development and Browser Support

Syncfusion<sup style="font-size:70%">&reg;</sup> Angular components take advantage of modern web standards, ensuring optimal performance and minimizing the need for polyfills or compatibility layers.

**Key Points:**

1. **Evergreen Browsers:** Components perform best with automatically updating browsers (Chrome, Firefox, Edge, Safari).

2. **Angular Compatibility:** Syncfusion<sup style="font-size:70%">&reg;</sup> components follow Angular's browser support policy.

3. **Mobile Support:** Full support for the latest versions of mobile browsers on iOS and Android platforms.

## Considerations for Internet Explorer 11

Internet Explorer 11 is supported with the following considerations:

- Some features may have limited functionality or require additional polyfills.
- Performance may be impacted compared to modern browsers.
- As IE11 usage continues to decline, consider the necessity of extensive IE11 support in new projects.

For additional details on polyfills and Angular browser support, refer to:
- [Angular Versions and Polyfills](https://angular.dev/reference/versions#actively-supported-versions)
